拆开看界面：

2. **数量：10% · USDT + 滑块**
   这里的百分比，是按下面那行的“**可开**”算的，意思是：

   > 用当前「最大可开仓」的大约 10% 作为本次下单的仓位大小。

3. 底部可以看到：

   * 左侧写着：`可开 24,986.532 USDT`
   * 10% × 24,986 ≈ 2,498 USDT，所以上面显示的买入数量就是约 **2,510 USDT**，基本就是“最大可开仓的 1/10”。

4. **保证金才是你真正占用的钱**
   同一块区域里还有一行：`保证金 124.86 USDT`
   这才是从你 5,000 USDT 里扣出来冻结的保证金。
   可以验证一下：

   * 仓位名义价值 / 杠杆 ≈ 保证金
   * 2,510 ÷ 20 ≈ 125 USDT，对上了。

总结：

* `买入 2,510.561 USDT` = 你这次想开的仓位大小（名义价值），约等于“最大可开仓”的 10%。
* `保证金 124.86 USDT` = 真正从 5,000 U 里占用的那一部分。
* 所以看起来像“明明选 10%，却是 2,500 多 U”，其实是因为百分比是对“最大可开仓”算的，不是对 5,000 U 余额算的。

# 什么是保证金？

下面按机制来讲，不按平台品牌区分。默认以 USDT 本位合约为例。

---

## 一、保证金是什么

在合约账户里，资金被拆成几块核心概念：

1. **钱包余额（Wallet Balance）**
   你充进合约账户的总资金，包含历史已实现盈亏。

2. **未实现盈亏（UPnL）**
   还没平仓的浮动盈亏，用“标记价格”计算。

3. **保证金相关**

   * **初始保证金 IM**：
     为开仓占用的那部分资金。
     [
     \text{初始保证金} = \frac{\text{合约名义价值}}{\text{杠杆倍数}}
     ]
   * **维持保证金 MM**：
     系统要求你至少要留的“底线保证金”。
     当账户净值跌到只够维持保证金时，就会触发强平。

4. **账户权益（Margin Balance / Equity）**
   [
   \text{账户权益} = \text{钱包余额} + \text{未实现盈亏}
   ]

爆仓判断本质是比较：

> 账户权益 是否还大于 维持保证金 + 手续费

---

## 二、开仓时保证金怎么占用

例子：

* 账户有 1,000 USDT
* 开 20 倍杠杆多仓，名义价值 10,000 USDT

1. 计算初始保证金
   [
   IM = 10,000 ÷ 20 = 500\ \text{USDT}
   ]
2. 钱包余额从 1,000 → 1,000（不变）
   只是其中 500 被标记为“仓位保证金”，不能随便用来下别的单。
3. 如果价格波动，你的未实现盈亏变化，账户权益变动：

   * 盈利 100

     * 未实现盈亏 = +100
     * 账户权益 = 1,000 + 100 = 1,100
   * 亏损 200

     * 未实现盈亏 = −200
     * 账户权益 = 800

是否爆仓取决于：

> 账户权益 是否还大于这笔仓位对应的维持保证金水平

---

## 三、全仓模式（Cross / 全仓保证金）

定义：

* 所有同一币种的可用资金**共享一个大池子**。
* 所有仓位和挂单都从这个池子里“共用保证金”。

### 1. 资金如何共享

假设：

* 账户有 1,000 USDT
* 开了两笔仓位（同是 USDT 本位）

仓位 A：BTC 多仓，名义价值 8,000 USDT
仓位 B：ETH 空仓，名义价值 4,000 USDT

在全仓模式下：

* 两个仓位的盈亏会一起反映在账户权益里
* 若 A 亏损，B 盈利，它们可以互相“对冲”，共同托底不被强平
* 强平判断用的是**整账户的权益**和**两仓位合计的维持保证金**

### 2. 全仓的强平特征

* 当总账户权益 ≈ 所有仓位的 MM + 手续费 时，系统开始逐步强平亏损最大的仓位。
* 如果还有其他仓位，可能被部分减仓或平掉，直到风险降到安全水平。

结果：

* 优点：有多笔仓位时，整体抗波动能力更强，爆仓价更远。
* 缺点：任何一单出问题，都可能把整个账户的资金一起拖下去。

你看到的“杠杆倍数”通常是一个**动态值**：
[
\text{有效杠杆} = \frac{\text{全部仓位名义价值总和}}{\text{账户权益}}
]
账户权益越大，有效杠杆越低，越安全。

---

## 四、逐仓模式（Isolated / 逐仓保证金）

定义：

* 每一条仓位都有**独立的小钱包**。
* 这条仓位爆掉，只亏掉分配给它的那部分保证金，不会默认动用其他余额。

### 1. 保证金如何隔离

还是刚才的条件：

* 账户有 1,000 USDT

你在逐仓模式里做：

* 仓位 A：BTC 多仓，分配 300 USDT 做保证金，开 15 倍杠杆

  * 名义价值 ≈ 4,500 USDT
* 仓位 B：ETH 空仓，分配 200 USDT 做保证金，开 10 倍杠杆

  * 名义价值 ≈ 2,000 USDT

特点：

* A 的风险只由这 300 + 它自己的浮盈浮亏决定
* B 的风险只由那 200 决定
* 账户剩余的 500 USDT 不会自动去给 A 或 B 补仓

你可以手动做两件事：

* 给某个逐仓仓位“追加保证金”
* 从某个仓位“抽回保证金”（在不影响安全线的前提下）

### 2. 逐仓爆仓机制

对每个仓位单独算：

[
\text{该仓位权益} = \text{分配保证金} + \text{该仓未实现盈亏}
]

当：

> 该仓位权益 ≤ 该仓位的维持保证金 + 手续费

就单独强平这条仓位。

结果：

* 优点：仓位之间互不牵连，可以明确“这单最多亏多少”。
* 缺点：哪怕账户还有钱，这条逐仓仓位也可能因为没有自动补仓，提前被强平。

---

## 五、全仓 vs 逐仓 对比总结

| 维度     | 全仓模式               | 逐仓模式                       |
| ------ | ------------------ | -------------------------- |
| 保证金来源  | 同一币种全部可用余额共享       | 每条仓位单独分配                   |
| 盈亏影响范围 | 一单大亏会拖累整个账户        | 一单爆了，只亏掉分配给它的那一部分          |
| 强平条件   | 基于账户整体权益与总维持保证金    | 基于单条仓位的权益与该仓维持保证金          |
| 爆仓价格   | 一般更远，抗临时波动更强       | 一般更近，但风险可控                 |
| 适用场景   | 单方向大仓位，或需要多仓位对冲    | 想给每个策略设定固定最大亏损，或短线试仓、网格策略等 |
| 人为操作   | 不需要频繁调保证金，只要关注整体风险 | 需要自己决定给每个仓位分配多少保证金，可能要手动追加 |

---

## 六、保证金、杠杆和仓位大小的关系

以逐仓为例，关系非常明确：

* 你设置杠杆倍数 `L`
* 你输入想开的**名义价值** `Notional`
* 则所需初始保证金：
  [
  IM = Notional ÷ L
  ]

反过来：

* 你先决定给这单分配 `IM` 保证金
* 设置杠杆 `L`
* 最大可开名义价值：
  [
  Notional_{\max} = IM × L
  ]

在全仓模式中：

* “保证金”不再固定在单仓位上
* 平台通常显示的是“总名义价值 ÷ 账户权益”的**有效杠杆**
* 一旦你在同一币种又新开仓，所有仓位共用同一池保证金，有效杠杆会整体变化

---

## 七、常见操作误区

1. 把“仓位大小”误当成“保证金”

   * 界面上显示的 2,000 USDT 往往是“名义价值”，真正占用保证金可能只有几十到几百。

2. 在全仓模式下，以为“只会亏这单的保证金”

   * 实际上爆仓时，系统会动用你账户里该币种的所有可用余额。

3. 在逐仓模式下忘记追加保证金

   * 明明账户还有很多钱，但因为仓位自己的保证金太小，被较早强平。

4. 忽视“维持保证金率”分档

   * 越大的仓位，平台给的维持保证金率越高，爆仓线会更靠近你。

# 如何计算爆仓？

判断“会不会爆仓”，核心就是看：

> **账户（或该仓位）的权益**
> 是否已经 ≤ **维持保证金**（再加一点费用缓冲）

下面给你一套可自己手算的逻辑，先给结论，再给公式。

---

## 1. 爆仓判断的通用逻辑

先分两种模式：

### 1）逐仓模式

对“单条仓位”计算：

* 该仓位权益
  [
  E_\text{pos} = \text{该仓位分配的保证金} + \text{该仓位未实现盈亏}
  ]

* 该仓位维持保证金
  [
  MM_\text{pos} = \text{仓位名义价值} \times \text{维持保证金率}
  ]

* 爆仓条件（忽略手续费、资金费）：

  > 当 (E_\text{pos} \le MM_\text{pos}) 时，这条仓位会被强平。

---

### 2）全仓模式

对“整账户”计算（同一币种）：

* 账户权益
  [
  E_\text{acc} = \text{钱包余额} + \sum \text{所有仓位未实现盈亏}
  ]

* 总维持保证金
  [
  MM_\text{total} = \sum \text{每条仓位的维持保证金}
  ]

* 爆仓条件：

  > 当 (E_\text{acc} \le MM_\text{total}) 时，系统开始按规则强平亏损大的仓位。

注意：这里用的价格都是“**标记价格**”，不是最新成交价。

---

## 2. 未实现盈亏怎么算？

以 USDT 本位合约为例，数量用“合约价值 / 价格”统一成 `Q`：

* 做多（Long）：
  [
  UPnL = (P_\text{mark} - P_\text{entry}) \times Q
  ]
* 做空（Short）：
  [
  UPnL = (P_\text{entry} - P_\text{mark}) \times Q
  ]

其中：

* (P_\text{mark})：当前标记价格
* (P_\text{entry})：你的开仓均价
* (Q)：合约数量（按币计）

仓位名义价值：

[
\text{Notional} = P_\text{mark} \times Q
]

维持保证金：

[
MM = \text{Notional} \times \text{维持保证金率 MMR}
]

MMR 是平台给出的一个很小的百分比（和仓位大小有关）。

---

## 3. 手算“是否会爆仓”的步骤

以“逐仓 + USDT 本位 + 做多”为例：

已知：

* 开仓价：(P_\text{entry})
* 当前标记价：(P_\text{mark})
* 合约数量：(Q)
* 分配给该仓的逐仓保证金：`M_init`（初始保证金+你追加的）
* 维持保证金率：`MMR`

步骤：

1. 算未实现盈亏
   [
   UPnL = (P_\text{mark} - P_\text{entry}) \times Q
   ]

2. 算该仓位权益
   [
   E_\text{pos} = M_\text{init} + UPnL
   ]

3. 算维持保证金
   [
   MM_\text{pos} = P_\text{mark} \times Q \times MMR
   ]

4. 比较：

   * 如果 (E_\text{pos} > MM_\text{pos})：暂时不会爆
   * 如果很接近：说明离爆仓不远
   * 若 (E_\text{pos} \le MM_\text{pos})：满足爆仓条件

在**全仓模式**下，把 `M_init` 换成“整个账户权益 `E_acc`”，把 `MM_pos` 换成“所有仓位 MM_total”，其余一样。

---

## 4. 反推“爆仓价格”（逐仓，近似公式）

很多人更关心“多少价位会爆”。

仍以 USDT 本位、逐仓、做多为例，假设：

* 杠杆：`L`
* 开仓价：(P_\text{entry})
* 数量：`Q`
* 初始保证金：
  [
  IM = \frac{P_\text{entry} \times Q}{L}
  ]
* 无额外追加保证金、忽略手续费和资金费
* 维持保证金率：`MMR`（固定）

在这种简化条件下，逐仓做多的爆仓价格大致是：

$$
P_\text{liq,long}
= \frac{P_\text{entry} \times (1 - \tfrac{1}{L})}{1 - MMR}
$$

逐仓做空的爆仓价大致是：

[
P_\text{liq,short}
= \frac{P_\text{entry} \times (1 + \tfrac{1}{L})}{1 + MMR}
]

平台实际显示的爆仓价会再扣手续费、资金费、阶梯维持保证金等，所以你手算只是近似。

---

## 5. 最实用的判断方式

数学上你可以按上面公式算。实战里更快的做法：

1. 在交易页面看“**保证金率 / 风险率**”

   * 通常显示为：
     [
     \text{保证金率} = \frac{MM}{E}
     ]
   * 接近 100% 就快爆了。

2. 或直接看平台给的“**强平价 / 爆仓价**”

   * 你只要看当前标记价离这个价还有多远。
