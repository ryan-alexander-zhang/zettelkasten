`mvn -q -DskipTests package -pl ddd-layout-template-adapter -am`

逐项解释：

* `mvn`: 执行 Maven 构建。
* `-q`: quiet 模式，减少日志输出，只保留关键信息和错误。
* `-DskipTests`: 跳过测试执行。测试代码仍会被编译（区别于 `-Dmaven.test.skip=true`，后者连测试编译也跳过）。
* `package`: 执行到 `package` 生命周期阶段。会编译、处理资源、运行（未跳过的）测试、打包成 jar 或 war 等。
* `-pl ddd-layout-template-adapter`: `--projects`，只选择这个模块参与构建（模块名通常是该模块的 artifactId 或在 reactor 中的模块坐标/路径）。
* `-am`: `--also-make`，同时构建 `-pl` 选中模块所依赖的其他本工程模块（reactor 内依赖），保证依赖模块先被构建。

综合效果：
只打包 `ddd-layout-template-adapter`，并自动把它依赖的本仓库其它模块一起打包，跳过测试，日志很少。
